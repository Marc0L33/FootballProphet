# Prophet v1.1.0 — محرك التنبؤ بمباريات كرة القدم

> [:us: English](../README.md) · [:cn: 中文](README_CN.md) · [:es: Español](README_ES.md) · [:brazil: Português](README_PT.md) · [:jp: 日本語](README_JA.md) · [:th: ไทย](README_TH.md) · [:ru: Русский](README_RU.md) · [:sa: العربية](README_AR.md)

محرك للتنبؤ بنتائج مباريات كرة القدم قائم على القواعد مع **إخراج مزدوج للنتيجة** (المنهجية + المعدلة حسب السوق)، **ثقة ثلاثية الأبعاد**، و**واجهة ويب أمامية** لتصفح المباريات المجمعة.

ينطبق على: **كأس العالم، البطولات القارية، الدوريات المحلية، الكؤوس، دوري الأبطال** — نفس سير العمل، مصادر بيانات مختلفة.

> دقة الاتجاه ~67%، النتيجة الدقيقة ~22%. تمت المعايرة عبر 72 مباراة من كأس العالم 2026 (بما في ذلك جميع مباريات الجولة الثالثة الـ18 عبر المجموعات D–L).

## البدء السريع

```bash
# 1. قم بإعداد ملف الإدخال (انظر test/spain_uruguay.json للقالب الكامل)
cat > match.json << 'EOF'
{
  "home_team": "South Africa", "away_team": "Canada",
  "home": {
    "tournament_matches": [{"gf":0,"ga":2},{"gf":1,"ga":1},{"gf":1,"ga":0}],
    "key_player_missing": [{"player":"Zwane","tier":"core_playmaker"}],
    "attacking_tier": 3, "rotation_count": 0
  },
  "away": {
    "tournament_matches": [{"gf":1,"ga":1},{"gf":6,"ga":0},{"gf":1,"ga":2}],
    "key_player_missing": [{"player":"Koné","tier":"core_playmaker"}],
    "attacking_tier": 2, "is_host": true, "rotation_count": 0
  },
  "match": {
    "tactical_matchup": {"home_advantage": "even", "away_advantage": "clear_advantage"},
    "venue_rain": false
  },
  "market": {
    "moneyline": {"home":5.00,"draw":3.40,"away":1.66},
    "handicap": "Canada -0.75",
    "totals": "Under 2.25",
    "signals": [{"type":"adjust_total","delta":-1,"reason":"U2.25 → مباراة خروج مغلول منخفضة الأهداف"}]
  }
}
EOF

# 2. تشغيل المحرك (مباراة واحدة)
python3 engine/predictor.py -i match.json

# 3. الوضع المجمع — معالجة جميع المباريات في مجلد
python3 engine/predictor.py --input output/predictions/ --out-dir output/web/

# 4. تشغيل الواجهة الأمامية
cd output/web && python3 -m http.server 8080
```

## نتيجتان، غرضان

يُخرج المحرك **نتيجتين جنبًا إلى جنب** بأدوار متمايزة:

| | نتيجة المنهجية | النتيجة المعدلة حسب السوق |
|---|---|---|
| **المصدر** | الصيغة البحتة (⑭+②+Σγδ) | الصيغة + إشارات السوق |
| **قابلية التكرار** | ✅ حتمية | ❌ تتطلب حكمًا نوعيًا |
| **الاستخدام** | خط الأساس؛ كشف انحراف المنهجية | مرجع عملي للمراهنة |
| **التوزيع** | يشارك λ المنهجية | نفس λ، نتيجة مدورة مختلفة |

عندما تتباعد النتيجتان، يعرف السوق شيئًا لا تعرفه المنهجية — الإصابات، الروح المعنوية، التواطؤ — لكن توزيع المنهجية يبقى مرتكزًا على واقع كرة القدم.

## صيغة التنبؤ

```
Goals_A = r14_blend_GF × r02_opponent_coeff + Σ(triggered_rule_i × γ_i × δ_i)
Goals_B = نفس الحساب، بشكل متماثل
Final Score = (round(Goals_A), round(Goals_B))
```

## التكامل مع Claude Code (SKILL.md)

هذا المشروع أيضًا **مهارة Claude Code**. سير العمل:

```
بحث (بيانات التصفيات، الإصابات، xG، الطقس، احتمالات المراهنة)
  → تدقيق البيانات (⑥ غسل، تحقق متبادل، تقسيم جولات AFC)
  → حكم نوعي (⑦ غياب، ⑫ تكتيكي، ⑲ تواطؤ)
  → 🔴 استخراج إشارات السوق (إلزامي، لا تترك فارغًا أبدًا)
  → تعبئة match.json → تشغيل المحرك → إخراج نتيجتين
```

التفويضات الرئيسية في SKILL.md:
- **أولوية المصادر الصينية** للتشكيلات، النتائج، البطاقات (懂球帝/虎扑/直播吧)
- **توقعات الطقس** يجب التحقق منها قبل المباراة (تفعّل ⑳)
- **إشارات السوق** يجب ألا تكون فارغة عند وجود بيانات الاحتمالات
- **⑲ + adjust_total التكديس ممنوع** (خلل العد الثلاثي تم التحقق منه بمباراة كرواتيا-غانا)

انظر [SKILL.md](../SKILL.md) لسير العمل الكامل.

## الواجهة الأمامية

`output/web/index.html` — عارض مباريات مجمع بالسحب والإفلات:
- رفع عدة ملفات JSON للتنبؤات
- تنقل بلوحة المفاتيح ← → بين المباريات
- عرض مزدوج للنتيجة (المنهجية | المعدلة حسب السوق)
- توزيعات احتمالية الأهداف لكل فريق مع رسوم بيانية شريطية
- أشرطة احتمالية الفوز/التعادل/الخسارة المشتركة
- شفافية التعديل حسب السوق
