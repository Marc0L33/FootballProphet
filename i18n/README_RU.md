# Prophet v1.1.0 — Футбольный Прогностический Движок

> [:us: English](../README.md) · [:cn: 中文](README_CN.md) · [:es: Español](README_ES.md) · [:brazil: Português](README_PT.md) · [:jp: 日本語](README_JA.md) · [:th: ไทย](README_TH.md) · [:ru: Русский](README_RU.md) · [:sa: العربية](README_AR.md)

Основанный на правилах движок прогнозирования футбольных матчей с **двойным выводом счёта** (методология + рыночная корректировка), **трёхмерной уверенностью** и **веб-интерфейсом** для пакетного просмотра матчей.

Применяется к: **ЧМ, континентальным турнирам, национальным лигам, кубкам, Лиге чемпионов** — единый рабочий процесс, разные источники данных.

> Точность направления ~67%, точный счёт ~22%. Откалибровано на 72 матчах ЧМ-2026 (включая все 18 матчей 3-го тура в группах D–L).

## Быстрый старт

```bash
# 1. Подготовьте входной файл (см. test/spain_uruguay.json для полного шаблона)
cat > match.json << 'EOF'
{
  "home_team": "South Africa", "away_team": "Canada",
  "home": {
    "tournament_matches": [{"gf":0,"ga":2},{"gf":1,"ga":1},{"gf":1,"ga":0}],
    "key_player_missing": [{"player":"Zwane","tier":"core_playmaker"}],
    "attacking_tier": 3, "rotation_count": 0
  },
  "away": {
    "tournament_matches": [{"gf":1,"ga":1},{"gf":6,"ga":0},{"gf":1,"ga":2}],
    "key_player_missing": [{"player":"Koné","tier":"core_playmaker"}],
    "attacking_tier": 2, "is_host": true, "rotation_count": 0
  },
  "match": {
    "tactical_matchup": {"home_advantage": "even", "away_advantage": "clear_advantage"},
    "venue_rain": false
  },
  "market": {
    "moneyline": {"home":5.00,"draw":3.40,"away":1.66},
    "handicap": "Canada -0.75",
    "totals": "Under 2.25",
    "signals": [{"type":"adjust_total","delta":-1,"reason":"U2.25 → низовая игра на вылет"}]
  }
}
EOF

# 2. Запуск движка (один матч)
python3 engine/predictor.py -i match.json

# 3. Пакетный режим — обработка всех матчей в директории
python3 engine/predictor.py --input output/predictions/ --out-dir output/web/

# 4. Запуск веб-интерфейса
cd output/web && python3 -m http.server 8080
```

## Два счёта, две цели

Движок выводит **два счёта рядом** с различными ролями:

| | Методологический счёт | Рыночно-скорректированный счёт |
|---|---|---|
| **Источник** | Чистая формула (⑭+②+Σγδ) | Формула + рыночные сигналы |
| **Воспроизводимость** | ✅ Детерминирован | ❌ Требует качественной оценки |
| **Применение** | Базовый уровень; обнаружение дрейфа методологии | Практический ориентир для ставок |
| **Распределение** | Использует λ методологии | Тот же λ, другой округлённый счёт |

Когда два счёта расходятся, рынок знает то, чего не знает методология — травмы, мораль, сговор — но распределение методологии остаётся основанным на футбольной реальности.

## Формула прогнозирования

```
Goals_A = r14_blend_GF × r02_opponent_coeff + Σ(triggered_rule_i × γ_i × δ_i)
Goals_B = то же вычисление, симметрично
Final Score = (round(Goals_A), round(Goals_B))
```

### Ключевые правила v1.1.0

| Правило | Что делает | Почему |
|------|-------------|-----|
| ㉑ Сила скамейки | Полностью компенсирует штраф r16 при bench ≥ 0.7 | Эра 5 замен: «отдохнувшие» звёзды играют 30 минут |

## Интеграция с Claude Code (SKILL.md)

Этот проект также является **навыком Claude Code**. Рабочий процесс:

```
Поиск (отборочные данные, травмы, xG, погода, коэффициенты ставок)
  → Аудит данных (⑥ очистка, перекрёстная проверка, разделение раундов AFC)
  → Качественная оценка (⑦ отсутствие, ⑫ тактика, ⑲ сговор)
  → 🔴 Извлечение рыночных сигналов (обязательно, никогда не оставлять пустым)
  → Заполнение match.json → Запуск движка → Вывод двух счётов
```

Ключевые требования в SKILL.md:
- **Приоритет китайских источников** для составов, счёта, карточек (懂球帝/虎扑/直播吧)
- **Прогноз погоды** должен проверяться перед матчем (активирует ⑳)
- **Рыночные сигналы** не должны быть пустыми при наличии данных коэффициентов
- **⑲ + adjust_total совмещение запрещено** (ошибка тройного учёта, проверено на Хорватия-Гана)

См. [SKILL.md](../SKILL.md) для полного рабочего процесса.

## Веб-интерфейс

`output/web/index.html` — просмотрщик матчей с перетаскиванием:
- Загрузка нескольких JSON с прогнозами
- Навигация клавишами ← → между матчами
- Двойной вывод счёта (методология | рыночная корректировка)
- Распределения вероятностей голов по командам с гистограммами
- Совместные столбцы вероятностей победа/ничья/поражение
- Прозрачность рыночной корректировки

```bash
cd output/web && python3 -m http.server 8080
# Откройте http://localhost:8080
```
