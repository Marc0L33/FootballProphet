# Prophet v1.1.0 — เครื่องมือทำนายผลการแข่งขันฟุตบอล

> [:us: English](../README.md) · [:cn: 中文](README_CN.md) · [:es: Español](README_ES.md) · [:brazil: Português](README_PT.md) · [:jp: 日本語](README_JA.md) · [:th: ไทย](README_TH.md) · [:ru: Русский](README_RU.md) · [:sa: العربية](README_AR.md)

เครื่องมือทำนายผลฟุตบอลตามกฎ พร้อม**เอาต์พุตสองคะแนน** (ตามวิธีการ + ปรับตามตลาด) **ความเชื่อมั่นสามมิติ** และ**เว็บฟรอนต์เอนด์**สำหรับดูผลการแข่งขันแบบกลุ่ม

ใช้ได้กับ: **ฟุตบอลโลก, ทัวร์นาเมนต์ระดับทวีป, ลีกในประเทศ, ถ้วย, แชมเปียนส์ลีก** — เวิร์กโฟลว์เดียวกัน แหล่งข้อมูลต่างกัน

> ความแม่นยำเชิงทิศทาง ~67%, สกอร์ตรง ~22% ปรับเทียบจาก 72 แมตช์ในฟุตบอลโลก 2026 (รวมทั้ง 18 แมตช์รอบสามในกลุ่ม D–L)

## เริ่มต้นใช้งาน

```bash
# 1. เตรียมไฟล์อินพุต (ดู test/spain_uruguay.json สำหรับเทมเพลตเต็ม)
cat > match.json << 'EOF'
{
  "home_team": "South Africa", "away_team": "Canada",
  "home": {
    "tournament_matches": [{"gf":0,"ga":2},{"gf":1,"ga":1},{"gf":1,"ga":0}],
    "key_player_missing": [{"player":"Zwane","tier":"core_playmaker"}],
    "attacking_tier": 3, "rotation_count": 0
  },
  "away": {
    "tournament_matches": [{"gf":1,"ga":1},{"gf":6,"ga":0},{"gf":1,"ga":2}],
    "key_player_missing": [{"player":"Koné","tier":"core_playmaker"}],
    "attacking_tier": 2, "is_host": true, "rotation_count": 0
  },
  "match": {
    "tactical_matchup": {"home_advantage": "even", "away_advantage": "clear_advantage"},
    "venue_rain": false
  },
  "market": {
    "moneyline": {"home":5.00,"draw":3.40,"away":1.66},
    "handicap": "Canada -0.75",
    "totals": "Under 2.25",
    "signals": [{"type":"adjust_total","delta":-1,"reason":"U2.25 → เกมแพ้คัดออกที่ทำประตูน้อย"}]
  }
}
EOF

# 2. รันเอนจิน (เดี่ยว)
python3 engine/predictor.py -i match.json

# 3. โหมดกลุ่ม — ประมวลผลทุกแมตช์ในโฟลเดอร์
python3 engine/predictor.py --input output/predictions/ --out-dir output/web/

# 4. เปิดเว็บฟรอนต์เอนด์
cd output/web && python3 -m http.server 8080
```

## สองคะแนน สองวัตถุประสงค์

เอนจินแสดงผล**สองคะแนนเคียงข้างกัน** โดยมีบทบาทต่างกัน:

| | คะแนนตามวิธีการ | คะแนนปรับตามตลาด |
|---|---|---|
| **แหล่งที่มา** | สูตรล้วน (⑭+②+Σγδ) | สูตร + สัญญาณตลาด |
| **ความสามารถในการทำซ้ำ** | ✅ แน่นอน | ❌ ต้องใช้ดุลยพินิจเชิงคุณภาพ |
| **การใช้งาน** | เส้นฐาน; ตรวจจับความคลาดเคลื่อนของวิธีการ | อ้างอิงเชิงปฏิบัติสำหรับการเดิมพัน |
| **การแจกแจง** | ใช้ λ ของวิธีการ | λ เดียวกัน คะแนนปัดเศษต่างกัน |

เมื่อสองคะแนนแตกต่างกัน แสดงว่าตลาดรู้บางสิ่งที่วิธีการไม่รู้ — การบาดเจ็บ ขวัญกำลังใจ การสมรู้ร่วมคิด — แต่การแจกแจงของวิธีการยังคงยึดตามความเป็นจริงของฟุตบอล

## สูตรการทำนาย

```
Goals_A = r14_blend_GF × r02_opponent_coeff + Σ(triggered_rule_i × γ_i × δ_i)
Goals_B = การคำนวณเดียวกัน แบบสมมาตร
Final Score = (round(Goals_A), round(Goals_B))
```

## การผสานรวมกับ Claude Code (SKILL.md)

โปรเจกต์นี้ยังเป็น**สกิล Claude Code** เวิร์กโฟลว์:

```
ค้นหา (ข้อมูลรอบคัดเลือก, การบาดเจ็บ, xG, สภาพอากาศ, อัตราต่อรอง)
  → ตรวจสอบข้อมูล (⑥ ล้าง, ตรวจสอบข้าม, แบ่งรอบ AFC)
  → การตัดสินเชิงคุณภาพ (⑦ ขาด, ⑫ กลยุทธ์, ⑲ สมรู้ร่วมคิด)
  → 🔴 ดึงสัญญาณตลาด (บังคับ ห้ามเว้นว่าง)
  → กรอก match.json → รันเอนจิน → แสดงผลสองคะแนน
```

ข้อกำหนดสำคัญใน SKILL.md:
- **ให้ความสำคัญแหล่งข้อมูลจีน** สำหรับรายชื่อผู้เล่น, สกอร์, ใบ (懂球帝/虎扑/直播吧)
- **พยากรณ์อากาศ** ต้องตรวจสอบก่อนแข่ง (กระตุ้น ⑳)
- **สัญญาณตลาด** ต้องไม่ว่างเมื่อมีข้อมูลอัตราต่อรอง
- **⑲ + adjust_total ห้ามซ้อนกัน** (ข้อผิดพลาดนับสามครั้ง ยืนยันจากโครเอเชีย-กานา)

ดู [SKILL.md](../SKILL.md) สำหรับเวิร์กโฟลว์เต็ม

## เว็บฟรอนต์เอนด์

`output/web/index.html` — เครื่องมือดูแมตช์แบบลากและวาง:
- อัปโหลดไฟล์ JSON ทำนายหลายไฟล์
- นำทางด้วยแป้นพิมพ์ ← → ระหว่างแมตช์
- แสดงสองคะแนน (วิธีการ | ปรับตามตลาด)
- การแจกแจงความน่าจะเป็นของประตูต่อทีมพร้อมกราฟแท่ง
- แท่งความน่าจะเป็นร่วม ชนะ/เสมอ/แพ้
- ความโปร่งใสในการปรับตามตลาด

```bash
cd output/web && python3 -m http.server 8080
# เปิด http://localhost:8080
```
